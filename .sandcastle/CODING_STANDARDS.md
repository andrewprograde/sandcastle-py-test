# Coding Standards

This file intentionally does **not** restate coding standards.

The canonical standards for any repository Sandcastle operates on live in that repo's
**`CONTRIBUTING.md`** (and its linter config, e.g. `pyproject.toml` `[tool.ruff]`). The
review prompt references those directly so there is a single source of truth and nothing
drifts out of sync.

If you are adapting Sandcastle for a repo with no `CONTRIBUTING.md`, add one there rather
than filling in standards here.
